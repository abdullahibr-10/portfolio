ICP — Unknown Data Association

Nov 2025 MSc Mobile Robotics, UniBonn

The full ICP algorithm deals with the real-world case: two point clouds of the same scene with no known correspondences. It alternates between finding the nearest-neighbour correspondences (data association) and computing the optimal rigid transform (alignment), repeating until convergence.

1. Full ICP Algorithm

Given source P and target Q with unknown correspondences, and an initial estimate (R₀, t₀):

Initialise: T = (R₀, t₀)

Repeat until convergence:
  1. DATA ASSOCIATION:
     For each pᵢ in P:
       q_i = argmin_{q ∈ Q} || q − (R pᵢ + t) ||
     (nearest neighbour search, e.g., k-d tree)

  2. ALIGNMENT:
     Solve SVD alignment (Lec 11) with correspondences
     {(pᵢ, qᵢ)} to get new (R*, t*)

  3. UPDATE: T ← (R*, t*) · T

Until: ||E_current − E_previous|| < threshold

2. Convergence Properties

ICP monotonically decreases the objective function at each iteration — the nearest-neighbour assignment can only decrease or maintain the cost, and the SVD alignment minimises cost for fixed correspondences. Therefore ICP always converges. However, it converges to a local minimum, which may not be the global minimum (correct alignment). The final result depends critically on initialisation.

3. Initialisation Sensitivity

ICP requires a reasonable initial alignment — typically within the "basin of attraction" of the correct solution. Common initialisation strategies:

  • Coarse-to-fine: subsample both point clouds to low resolution, run ICP, then use the result as initialisation for the full-resolution ICP.
  • Feature-based: extract FPFH (Fast Point Feature Histograms) descriptors at keypoints, match, compute initial transform via RANSAC + SVD. This global registration does not require a good initial guess.
  • 4PCS (Four-Point Congruent Sets): finds a congruent set of 4 coplanar points in both scans — a global registration method for no-initial-guess scenarios.

4. Robust ICP

Standard nearest-neighbour correspondences are sensitive to outliers — a noisy point or a scan from a partially overlapping region will corrupt correspondences. Solutions:

  • Distance threshold: reject pairs where ||qᵢ − Rpᵢ − t|| > d_max. Discards points with no nearby match in the target (likely in non-overlapping regions).
  • Trimmed ICP: only use the closest s% of pairs (e.g., s = 80%). Automatically adapts to partial overlap without a fixed threshold.
  • Robust kernels: replace squared distance with Huber or Cauchy loss — large residuals (outliers) are down-weighted instead of discarded.

5. EM Interpretation

ICP can be viewed as EM for a mixture model where each source point is a Gaussian centred on its nearest target neighbour. The E-step (nearest-neighbour) performs hard assignment (one Gaussian gets all the probability for each source point). The M-step (SVD alignment) maximises the likelihood given those assignments. Soft-assign ICP (EM-ICP) replaces hard assignment with soft Gaussian weights, giving smoother convergence behaviour and better handling of ambiguous nearest neighbours.

6. Generalised ICP (GICP)

GICP (Segal et al., 2009) models each point as a Gaussian distribution with local covariance Cᵢ estimated from its neighbourhood. This captures the local surface geometry: points on a flat plane have a thin, disc-shaped covariance; points on a sharp edge have a thin, rod-shaped covariance. The alignment minimises the Mahalanobis distance between corresponding Gaussians, effectively implementing plane-to-plane registration. GICP achieves better accuracy and faster convergence than standard point-to-point or point-to-plane ICP, and is the state-of-the-art for LiDAR scan matching.

ICP in ROS In ROS2, point cloud registration can be done with the PCL library's pcl::IterativeClosestPoint (point-to-point), pcl::IterativeClosestPointWithNormals (point-to-plane), or the ndt_scan_matcher package (NDT — Normal Distribution Transform, an alternative to ICP). For GICP, use fast_gicp or hdl_graph_slam.
