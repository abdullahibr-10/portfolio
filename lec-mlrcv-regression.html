Regression

Oct 2025 MSc Mobile Robotics, UniBonn

Regression is the foundational supervised learning task: given labelled training pairs (x, y), learn a mapping that generalises to unseen inputs. In robotics, regression appears everywhere — estimating continuous joint torques, predicting depth from image pixels, or fitting sensor calibration curves.

1. Supervised Learning Setup

We have a training set of m examples {(x⁽ⁱ⁾, y⁽ⁱ⁾)} where x ∈ ℝⁿ is the feature vector and y ∈ ℝ is the continuous target. Our goal is to learn a hypothesis h_θ(x) parameterised by θ such that h_θ(x) ≈ y for new inputs. The quality of this approximation is measured by a loss function, typically chosen to reflect the task's error sensitivity.

2. Linear Regression

The simplest hypothesis is a linear function h_θ(x) = θᵀx (with a bias term absorbed via an augmented feature x₀ = 1). The standard cost function is the mean squared error:

J(θ) = (1/2m) Σᵢ (h_θ(x⁽ⁱ⁾) − y⁽ⁱ⁾)²

For linear regression the minimiser of J(θ) has a closed-form solution called the Normal Equation:

θ = (XᵀX)⁻¹ Xᵀy

where X is the m×(n+1) design matrix. This is exact but costs O(n³) to invert — impractical when n is large (e.g., raw pixel features). In those cases, iterative optimisation is preferred.

3. Gradient Descent

Gradient descent minimises J(θ) iteratively. Starting from an initial θ, repeatedly apply the update rule:

θⱼ ← θⱼ − α · (∂J/∂θⱼ) = θⱼ − (α/m) Σᵢ (h_θ(x⁽ⁱ⁾) − y⁽ⁱ⁾) xⱼ⁽ⁱ⁾

The learning rate α controls step size. Too large: overshoots and diverges. Too small: very slow convergence. Stochastic gradient descent (SGD) updates using one sample at a time; mini-batch uses a small batch (32–256) — a compromise between noise and computation. Convergence is guaranteed for convex J (such as MSE with linear h_θ).

4. Polynomial Regression

To capture non-linear relationships, we augment features with polynomial terms: for a scalar input x, form φ(x) = [1, x, x², …, xᵈ]ᵀ and apply linear regression in this higher-dimensional feature space. The hypothesis h_θ(φ(x)) can fit complex curves while remaining linear in θ. However, high-degree polynomials (d ≫ 1) will fit training noise — a phenomenon called overfitting — and generalise poorly.

5. Regularisation

Regularisation adds a penalty on the magnitude of θ to discourage overfitting:

  • Ridge (L2): J_ridge(θ) = J(θ) + λ Σⱼ θⱼ². This shrinks all weights towards zero smoothly. The normal equation becomes θ = (XᵀX + λI)⁻¹Xᵀy, which also fixes the rank-deficiency of XᵀX.
  • Lasso (L1): J_lasso(θ) = J(θ) + λ Σⱼ |θⱼ|. The non-differentiable penalty at zero produces sparse solutions — many θⱼ are driven to exactly zero, performing automatic feature selection.

The regularisation strength λ is a hyperparameter: large λ increases bias, small λ allows overfitting. It is selected by cross-validation.

6. Cross-Validation

In k-fold cross-validation, the training set is split into k equal folds. For each fold i, the model trains on the remaining k−1 folds and is evaluated on fold i. The average validation error across all folds estimates generalisation performance. We sweep over candidate hyperparameters (λ, polynomial degree) and select the configuration with the lowest cross-validated error. k = 5 or 10 is typical.

7. Bias-Variance Tradeoff

The expected test error decomposes as: Bias² + Variance + Irreducible Noise. A too-simple model (low d, high λ) has high bias — it systematically underfits the data, producing large training and test error. A too-complex model (high d, low λ) has high variance — it fits training noise and performs much worse on test data than on training data. Optimal model complexity minimises the total error, balancing these two sources.

Key Takeaway Regularisation and cross-validation are inseparable: always tune λ on a held-out validation set, never on the test set. Reporting test error on the same data used for hyperparameter selection produces overly optimistic estimates.

Regression provides the scaffolding for more advanced ML: logistic regression adapts this framework for classification by passing h_θ(x) through a sigmoid, and neural network output layers often use the same MSE or cross-entropy cost functions.
