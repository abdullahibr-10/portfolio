Classification

Oct 2025 MSc Mobile Robotics, UniBonn

Classification assigns discrete labels to inputs. In robotics this covers terrain type recognition, object category prediction, fault detection, and semantic segmentation. Understanding the strengths and failure modes of each classifier is crucial for choosing the right tool for a given task.

1. Binary Classification Setup

Given input x ∈ ℝⁿ and label y ∈ {0, 1}, we learn a decision function f(x) that predicts the class. The decision boundary is the surface in feature space where f(x) = 0.5 (or the threshold), separating predicted class 0 from class 1. The geometry of this boundary — linear, curved, piecewise — determines the expressive power of the classifier.

2. Logistic Regression

Logistic regression models the conditional probability P(y=1 | x) = σ(θᵀx), where σ is the sigmoid function:

σ(z) = 1 / (1 + e−z)

Output is bounded in (0, 1) and interpreted as a probability. Training minimises the cross-entropy loss:

J(θ) = −(1/m) Σᵢ [ y⁽ⁱ⁾ log h_θ(x⁽ⁱ⁾) + (1−y⁽ⁱ⁾) log(1−h_θ(x⁽ⁱ⁾)) ]

This is convex in θ, so gradient descent converges to the global minimum. The decision boundary is linear: θᵀx = 0.

3. k-Nearest Neighbours

k-NN is a non-parametric method: to classify a new point x, find the k closest training points by some distance metric (Euclidean, cosine) and take a majority vote. It requires no explicit training but stores the entire dataset. Key considerations:

  • Small k: low bias, high variance (noisy boundaries).
  • Large k: smooth boundary, but may blur near true decision boundaries.
  • Curse of dimensionality: in high dimensions, all points become approximately equidistant. k-NN degrades unless features are carefully engineered or dimensionality is reduced first.

4. Support Vector Machines

SVMs find the maximum-margin hyperplane — the linear boundary that maximises the distance to the nearest training points (the support vectors). For linearly separable data, the primal optimisation is:

min (1/2)||w||² subject to y⁽ⁱ⁾(wᵀx⁽ⁱ⁾ + b) ≥ 1 for all i

The kernel trick implicitly maps features to a high-dimensional space via a kernel function k(x, x') = φ(x)ᵀφ(x'), allowing non-linear boundaries without computing φ explicitly. Common kernels: RBF k(x,x') = exp(−γ||x−x'||²) and polynomial k(x,x') = (xᵀx' + c)ᵈ.

5. Multi-Class Classification

For K > 2 classes, two common strategies are:

  • One-vs-Rest (OvR): train K binary classifiers, each separating class k from all others. Predict the class with the highest confidence score.
  • Softmax (multinomial logistic): model P(y=k | x) = exp(θₖᵀx) / Σⱼ exp(θⱼᵀx). Directly models the joint distribution over K classes and is differentiable end-to-end — the output layer of most neural networks for classification.

6. Evaluation Metrics

Accuracy alone is misleading for imbalanced datasets. A full evaluation uses the confusion matrix, from which we derive:

  • Precision = TP / (TP + FP): of predicted positives, how many are correct?
  • Recall = TP / (TP + FN): of actual positives, how many did we detect?
  • F1 score = 2 · Precision · Recall / (Precision + Recall): harmonic mean, balances both.
  • ROC/AUC: plot True Positive Rate vs False Positive Rate across all thresholds. Area Under the Curve (AUC = 1 is perfect, 0.5 is random) summarises performance across operating points.
Robotics Perspective In safety-critical tasks (e.g., obstacle detection), recall matters more than precision — missing a positive is worse than a false alarm. Always align your evaluation metric with the task's cost asymmetry before selecting a classifier.

The classifiers covered here — logistic regression, k-NN, SVM — form a useful toolkit. Deep learning has superseded them for raw image inputs, but they remain relevant for structured features, small datasets, and interpretability-critical applications.
