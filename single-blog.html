Bayes Filters and the Recursive State Estimator

Muhammad Abdullah October 2025 MSc Mobile Robotics, UniBonn

The central problem in mobile robotics is state estimation: given noisy sensor data and imperfect actuators, what is our best belief about the robot's current state? The Bayes filter provides a principled, recursive answer.

Source Thrun, Burgard & Fox — Probabilistic Robotics (2005), Chapter 2. Lecture slides from Prof. Cyrill Stachniss, University of Bonn.

1. The Robot State Estimation Problem

We model the world as a discrete-time dynamical system. At each time step t, the robot occupies some state x_t (e.g., pose in 2D space). We cannot observe the state directly — we receive observations z_t from sensors and apply control inputs u_t.

The belief bel(x_t) is a probability distribution over all possible states at time t, conditioned on all past observations and controls:

bel(xt) = p(xt | z1:t, u1:t)

2. The Markov Assumption

The Bayes filter relies on the Markov assumption: the current state x_t is a complete summary of all past information. Given x_t, the future is conditionally independent of the past. This allows us to write two models:

  • Motion model p(x_t | u_t, x_{t-1}): probability of transitioning to x_t given previous state and control.
  • Observation model p(z_t | x_t): probability of receiving measurement z_t given state x_t.

3. The Bayes Filter Algorithm

The filter alternates between two steps. For every state x_t: 

Algorithm Bayes_Filter(bel(x_{t-1}), u_t, z_t):

  -- PREDICTION STEP (integrate motion)
  for all x_t:
    bel_bar(x_t) = ∫ p(x_t | u_t, x_{t-1}) · bel(x_{t-1}) dx_{t-1}

  -- CORRECTION STEP (integrate observation)
  for all x_t:
    bel(x_t) = η · p(z_t | x_t) · bel_bar(x_t)

  return bel(x_t)

Here η is a normalisation constant ensuring bel(x_t) integrates to 1. The prediction step propagates the belief through the motion model (causing spread/uncertainty increase). The correction step weights each state by how well the current observation matches.

Intuition

Think of bel(x_t) as a "possibility map" over where the robot might be. Moving causes this map to blur (higher uncertainty). Observing a landmark causes it to sharpen around states that are consistent with the measurement.

Key Insight The Bayes filter is a framework, not an algorithm. Kalman Filters, Particle Filters, Histogram Filters are all concrete instantiations for different state/noise assumptions. The choice of representation determines the tractability and accuracy of the filter.

4. When Does This Break?

The Markov assumption fails when the state is not fully observable — i.e., important world features are not modeled. Classic failure cases in mobile robotics:

  • Dynamic obstacles not included in state (moving people, cars)
  • Sensor aliasing — identical observations from different locations
  • Unmodeled actuator lag or calibration error in the motion model

In practice the assumption is never perfectly satisfied, but the filter degrades gracefully if the unmodeled effects are small.

5. Summary

The Bayes filter is the theoretical backbone of probabilistic robotics. It gives us a recursive formula to maintain a belief over robot state by alternating motion prediction and measurement correction. All practical filters (KF, EKF, UKF, PF) are derived from this framework under specific assumptions about the state space and noise distributions.
