3D World Representations

Oct 2025 MSc Mobile Robotics, UniBonn

A humanoid operating in the real world must maintain an internal model of the 3D environment for both navigation and manipulation. The choice of world representation is a key engineering decision that affects memory usage, update speed, query efficiency, and compatibility with downstream planning algorithms.

1. Why 3D Matters for Humanoids

Unlike wheeled robots, humanoids interact with the world volumetrically: they step over objects, reach around obstacles, and grasp items from arbitrary angles. A 2D occupancy grid (flat-world assumption) is insufficient — the robot must know the 3D shape of objects to plan collision-free arm motions and to choose appropriate grasp poses. Navigation requires modelling the 3D free space through which the legs and body must pass.

2. Point Clouds

A point cloud is an unordered set of 3D points {xᵢ} ∈ ℝ³, typically obtained directly from RGB-D cameras or LiDAR. Each point may also carry colour (RGB) and intensity (from LiDAR). Advantages: direct sensor output, no discretisation error, compact for sparse environments. Disadvantages: no explicit connectivity between points, surface normals must be estimated from neighbourhoods, queries (collision checking, nearest neighbour) require spatial indexing structures (k-d tree, octree). The PCL (Point Cloud Library) provides standard algorithms for filtering, segmentation, and registration on point clouds.

3. Triangle Meshes

A triangle mesh represents a surface as a set of vertices V and triangular faces F. Meshes are compact (memory-efficient for smooth surfaces), support per-vertex normals and texture coordinates, and are the standard format for physics simulation (MuJoCo, PyBullet, Gazebo) and collision checking (FCL library). However, meshes are hard to update online — adding new observations requires re-meshing, which is expensive. They are therefore mainly used for pre-known object models, not for dynamic environment maps.

4. Signed Distance Fields (SDF)

An SDF stores at each point in space the signed distance to the nearest surface: positive outside, negative inside, zero on the surface. SDFs enable:

  • Smooth collision checking: the SDF value at a query point is the minimum clearance to any obstacle. Zero-crossing = collision.
  • Gradient-based motion planning: the gradient of the SDF at any point always points away from the surface — useful for gradient-based obstacle avoidance (CHOMP, GPMP).
  • Surface extraction: marching cubes on the SDF produces a triangle mesh of the surface.

5. Truncated SDF (TSDF)

KinectFusion (Newcombe et al., 2011) introduced the TSDF for real-time dense 3D reconstruction. Only store the SDF within a narrow band (±truncation distance τ) around the surface. Each voxel stores a running weighted average of SDF observations: d_new = (w_old · d_old + w_obs · d_obs) / (w_old + w_obs). Observations come from depth image ray casting. The result is a complete, noise-averaged surface model that fuses hundreds of depth frames. Marching cubes extracts the mesh for visualisation or manipulation planning.

6. OctoMap

OctoMap (Hornung et al., 2013) is the standard for 3D navigation maps in ROS. It uses an octree (recursive spatial subdivision) for memory-efficient volumetric occupancy. Each leaf voxel stores a log-odds occupancy probability. Updates use ray casting from the sensor: occupied cells at measurement endpoints increment their log-odds; free cells along the ray decrement. Key advantages:

  • Memory-adaptive: only allocate voxels for occupied or free space; unknown space uses no memory.
  • Multi-resolution: query at different resolutions by traversing to different tree depths.
  • Probabilistic: integrates uncertain measurements correctly, handles moving objects via temporal decay.

7. RGB-D Sensors

Practical depth sensors for humanoid manipulation:

  • Structured light (Kinect v1, Intel RealSense D435): project a known IR pattern and decode distortion to compute depth. Good accuracy at 0.5–3m, fails outdoors (sunlight swamps IR).
  • Stereo (ZED 2, RealSense D435): passive stereo — works outdoors, accuracy degrades in textureless regions.
  • Time-of-flight (ToF, Kinect v2): measure round-trip time of IR pulses. More accurate than structured light, better outdoors, but higher noise at range.
Representation Choice for the G1 For the Unitree G1 lab: OctoMap for whole-room navigation planning; TSDF (using Open3D or Voxblox) for fine-resolution manipulation workspace modelling around the hands; point clouds as the raw sensor output and for real-time obstacle checking.
