ICP — Known Data Association

Nov 2025 MSc Mobile Robotics, UniBonn

Iterative Closest Point (ICP) is the workhorse algorithm for aligning 3D point clouds. It is used in LiDAR odometry, map building, and object pose estimation. This lecture addresses the simpler case where the point-to-point correspondences are already known — the core SVD-based alignment step.

1. Problem Setup

Given a source point set P = {p₁, ..., pₙ} and a target point set Q = {q₁, ..., qₙ} with known correspondences (pᵢ ↔ qᵢ), find the rigid transform (R, t) — R ∈ SO(3), t ∈ ℝ³ — that minimises:

E(R, t) = Σᵢ || qᵢ − (R pᵢ + t) ||²

This is the least-squares rigid registration problem. It has a closed-form solution via SVD, avoiding iterative optimisation for this sub-problem.

2. SVD-Based Closed-Form Solution

Given: P = {p₁..pₙ}, Q = {q₁..qₙ}, known correspondences pᵢ ↔ qᵢ

1. Compute centroids:
   p̄ = (1/n) Σ pᵢ,   q̄ = (1/n) Σ qᵢ

2. Centre the point sets:
   p'ᵢ = pᵢ − p̄,   q'ᵢ = qᵢ − q̄

3. Compute cross-covariance matrix:
   H = Σᵢ p'ᵢ q'ᵢᵀ    (3×3 matrix)

4. SVD:  H = U Σ Vᵀ

5. Compute rotation:
   R = V Uᵀ
   If det(R) = −1:  V[:,2] = −V[:,2], recompute R
   (handles reflection ambiguity in degenerate configs)

6. Compute translation:
   t = q̄ − R p̄

The reflection fix in step 5 is important — SVD can produce a reflection (det = −1) in degenerate cases (e.g., all points coplanar). Negating the last column of V corrects this.

3. Why This Works

The key insight is that the optimal translation, given the optimal rotation, is t* = q̄ − R*p̄ (aligns centroids). Substituting this back, the cost reduces to minimising Σᵢ ||q'ᵢ − R p'ᵢ||², which is equivalent to maximising trace(RH). The SVD of H maximises this trace, and R = VUᵀ is the solution. The derivation uses the identity trace(RH) = trace(VᵀRU Σ) and the fact that VᵀRU ∈ SO(3) with trace ≤ 3 (achieved at identity).

4. Point-to-Point vs Point-to-Plane

Point-to-point: minimises Σᵢ ||qᵢ − Rpᵢ − t||² — the 3D Euclidean distance between corresponding points. Simple and general, but converges slowly (linear rate) on smooth surfaces because many nearby points have similar point-to-point distances.

Point-to-plane: minimises the distance along the surface normal at each target point: Σᵢ (nᵢᵀ(qᵢ − Rpᵢ − t))². Projects the error onto the surface normal direction. Requires normals at target points (estimated from the neighbourhood). Converges much faster (super-linear) on planar/smooth surfaces because it correctly models surface geometry — a slide along the surface costs nothing.

5. Applications

  • LiDAR odometry: LOAM and LeGO-LOAM use point-to-plane ICP on edge and planar features extracted from LiDAR scans to estimate 6-DOF odometry at 10 Hz.
  • Scan matching in 2D SLAM: align successive 2D laser scans to estimate robot motion.
  • Object pose estimation: given a known 3D model and a partial scan, recover the 6-DOF pose.
  • Loop closure: verify a candidate loop closure by registering the current scan to a stored map keyframe.
Known vs Unknown DA This lecture assumed correspondences are given. In practice, correspondences are unknown — we only have two point clouds.
