Fundamental & Essential Matrix

Oct 2025 MSc Mobile Robotics, UniBonn

The fundamental matrix F encodes the geometric relationship between two camera views without requiring knowledge of the intrinsic parameters. It constrains where corresponding points can appear across images, enabling feature matching verification and relative pose recovery.

1. The Epipolar Constraint

Given a point x in image 1, its corresponding point x' in image 2 is not arbitrary — it must lie on a specific line l' called the epipolar line. The fundamental matrix F encodes this relationship: l' = Fx. The constraint that x' lies on l' gives the fundamental epipolar constraint:

x'ᵀ F x = 0

This scalar equation must hold for every true correspondence (x, x'). It is a necessary (but not sufficient) condition for a point pair to be a correct match — it can be used to filter mismatches.

2. Properties of the Fundamental Matrix

F is a 3×3 matrix with the following properties:

  • Rank 2: det(F) = 0. This enforces that the epipolar lines in both images all pass through the respective epipoles.
  • 7 DOF: 9 elements, minus 1 for scale (homogeneous), minus 1 for the rank-2 constraint = 7 degrees of freedom.
  • F depends only on the relative pose between cameras and the intrinsic parameters — not on the 3D scene.

3. The 8-Point Algorithm

Given n ≥ 8 point correspondences, each pair (xᵢ, x'ᵢ) provides one linear equation in the 9 entries of F via x'ᵀFx = 0. Stack n constraints into Af = 0 where f = vec(F) and A is n×9.

  1. Solve Af = 0 via SVD: f is the last column of V in A = UΣVᵀ.
  2. Reshape f to 3×3 to get F̂.
  3. Enforce rank-2 constraint: compute SVD F̂ = UΣVᵀ, set the smallest singular value to 0, reconstruct: F = U diag(σ₁, σ₂, 0) Vᵀ.

This gives the algebraically optimal F for the given correspondences (without normalisation — see next lecture for why normalisation is critical).

4. The Essential Matrix

For calibrated cameras (K known for both cameras), we work in normalised image coordinates x̄ = K⁻¹x. The essential matrix E is the fundamental matrix in these coordinates: E = KᵀFK'. It relates to the relative camera pose by E = [t]× R, where R is the relative rotation and [t]× is the skew-symmetric cross-product matrix of the translation vector t.

Key properties of E:

  • 5 DOF: 3 for rotation (SO(3)), 2 for translation direction (scale unknown in monocular vision) = 5 DOF.
  • Singular values of E satisfy σ₁ = σ₂ > 0, σ₃ = 0 (two equal non-zero singular values). This more rigid structure makes E more stable to estimate than F when K is known.

5. Recovering [R | t] from E

Given E = UΣVᵀ, there are four candidate decompositions into (R, t):

  • R = U W Vᵀ or R = U Wᵀ Vᵀ (where W = [[0,-1,0],[1,0,0],[0,0,1]])
  • t = ±u₃ (the third column of U)

This yields 4 combinations. The correct solution is selected via the chirality check: triangulate a point from both cameras using each candidate (R, t) and select the solution where the reconstructed point lies in front of both cameras (positive depth in both camera frames). In practice, only one of the four solutions satisfies this.

F vs E in Practice Use E when cameras are calibrated (known K) — it has fewer DOF, is more constrained, and gives more accurate pose recovery. Use F when intrinsics are unknown or unreliable, but note that you cannot recover the metric pose from F alone, only the projective structure.
