Introduction to CNNs

Nov 2025 MSc Mobile Robotics, UniBonn

Convolutional Neural Networks (CNNs) are the dominant architecture for image-based perception. They exploit the spatial structure of images through local connectivity and weight sharing, enabling efficient learning of hierarchical features without manual design.

1. Why Not Fully-Connected Layers for Images?

A fully-connected (FC) layer connects every input unit to every output unit. For a modest 224×224×3 colour image, the input has 150,528 values. A single FC layer with 4,096 units would have 150,528 × 4,096 ≈ 617 million parameters — just for one layer. This is computationally prohibitive, and ignores the key structure of images: nearby pixels are correlated, and the same local pattern (an edge, a corner) can appear anywhere in the image. FC layers learn no such inductive biases.

2. The Convolutional Layer

A convolution layer applies a small learnable filter (kernel) K of size k×k across the entire input feature map F. The output (feature map) at position (i, j) is:

(F ★ K)(i, j) = Σₘ Σₙ F(i+m, j+n) · K(m, n)

Key hyperparameters:

  • Kernel size: typically 3×3 or 5×5. Smaller kernels have fewer parameters and allow deeper networks for the same receptive field.
  • Stride s: how many pixels the kernel moves per step. Stride 2 halves the spatial dimensions.
  • Padding: "valid" padding shrinks the output; "same" padding adds zeros around the border so the output has the same H×W as the input.

A layer with F filters applied to an input of shape H×W×C produces an output of shape H'×W'×F. Each filter is a C-channel learnable weight tensor — so a 3×3×C filter has 9C parameters + 1 bias.

3. Parameter Sharing

The same filter weights are applied at every spatial position — this is parameter sharing. A 3×3×3 filter has only 27 learnable weights regardless of the input image size. Compare to an FC layer connecting 32×32×3 input to a single 32×32 output map: 32×32×3 × 32×32 = ~3 million weights. Parameter sharing exploits translation invariance: an edge detector should look the same wherever edges appear.

4. Pooling Layers

Pooling downsamples the spatial dimensions, reducing computation and providing local translation invariance. Max pooling takes the maximum value in each pooling window (e.g., 2×2 with stride 2 halves H and W). Average pooling takes the mean. Max pooling is more common in classification networks because it selects the strongest activation, discarding weak responses. Global average pooling (GAP) averages each entire feature map to a scalar — used in modern architectures (ResNet, MobileNet) to replace FC layers before the output.

5. Activation Functions

Without a non-linearity, stacking conv layers is equivalent to a single linear transformation. The Rectified Linear Unit (ReLU) is the standard choice:

ReLU(x) = max(0, x)

ReLU advantages: (1) no vanishing gradient for positive activations — the gradient is exactly 1; (2) computationally trivial; (3) induces sparsity (negative activations become 0). The "dying ReLU" problem (units permanently outputting 0 if bias becomes very negative) is addressed by Leaky ReLU and ELU variants.

6. LeNet-5 — Historical Architecture

LeCun et al.'s LeNet-5 (1998) pioneered the modern CNN design for handwritten digit recognition: CONV(6 filters, 5×5) → AvgPool(2×2) → CONV(16 filters, 5×5) → AvgPool(2×2) → FC(120) → FC(84) → Output(10). It achieved 99.2% on MNIST with only ~60k parameters. The core insight — alternating convolution and pooling layers to build increasingly abstract representations — remains the backbone of all modern CNNs.

Receptive Field The receptive field of a unit is the region of the input that can influence its output. Stacking conv layers with 3×3 kernels: layer 1 sees 3×3, layer 2 sees 5×5, layer 3 sees 7×7. Deep networks have large effective receptive fields despite using small kernels — each 3×3 layer adds 2 to the effective receptive field size.
