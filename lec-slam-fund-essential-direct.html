Fundamental & Essential Matrix — Direct Methods

Oct 2025 MSc Mobile Robotics, UniBonn

The previous lecture derived the 8-point algorithm as a DLT applied to point correspondences. This lecture addresses its practical failure modes and the robust estimation pipeline that makes F/E estimation usable in real systems with noisy feature matches and outliers.

1. Why Raw 8-Point Fails

Pixel coordinates of detected features typically lie in the range [0, 1920] × [0, 1080] for an HD image. When these are placed in matrix A, rows differ by orders of magnitude. The condition number of A becomes enormous, making the SVD numerically unreliable — small changes in the input (noise of even 1 pixel) can lead to large changes in the solution. The resulting F will have poor geometric accuracy even with many inlier correspondences.

2. Hartley Normalisation

The fix is to normalise both point sets before constructing A. For each image independently:

  1. Compute the centroid of all points: μ = (1/n) Σ xᵢ.
  2. Translate so centroid is at the origin: x'ᵢ = xᵢ − μ.
  3. Compute the mean distance from the origin: d̄ = (1/n) Σ ||x'ᵢ||.
  4. Scale uniformly: x̃ᵢ = (√2 / d̄) · x'ᵢ, so that mean distance from origin is exactly √2.

This transformation can be written as a 3×3 matrix T (for image 1) and T' (for image 2). After solving the normalised system for F_norm, denormalise: F = T'ᵀ F_norm T. This simple step dramatically improves numerical conditioning and therefore accuracy of the estimated F.

3. RANSAC for Robust Estimation

Even with good feature detectors, matching across two images produces many wrong correspondences (outliers). The ratio of outliers ε can be 50% or higher in scenes with repeated textures or significant viewpoint change. Running the 8-point algorithm on the full set would give a corrupted F.

RANSAC (Random Sample Consensus) iteratively samples the minimum number of correspondences needed to fit the model, scores on all data, and keeps the best solution:

  1. Sample 8 random correspondences.
  2. Compute F using the normalised 8-point algorithm.
  3. Count inliers: correspondences where the Sampson distance < threshold τ (typically 1–3 pixels).
  4. Repeat N iterations. Keep the F with the most inliers.

The required number of iterations is:

N = log(1 − p) / log(1 − (1 − ε)⁸)

where p is the desired probability of finding at least one all-inlier sample (typically p = 0.99) and ε is the outlier fraction. For ε = 0.5, N ≈ 1177 iterations.

4. Sampson Distance

The Sampson distance is a first-order approximation of the reprojection error and is much cheaper to compute than the exact geometric error. For a correspondence (x, x'):

d_S(x, x')² = (x'ᵀFx)² / ( (Fx)₁² + (Fx)₂² + (Fᵀx')₁² + (Fᵀx')₂² )

The numerator (x'ᵀFx)² is the squared epipolar constraint violation. The denominator normalises by the squared norms of the epipolar lines, giving a distance in pixel units. Points with d_S < τ are classified as inliers.

5. Gold Standard Refinement

After RANSAC identifies inliers, refine F on all inliers by minimising the sum of Sampson distances using Levenberg-Marquardt (LM). This Gold Standard method gives the maximum likelihood estimate of F under Gaussian pixel noise. In practice:

  • Parameterise F to enforce rank-2 (7 parameters) for the optimisation.
  • Use the RANSAC result as initialisation.
  • Run LM to convergence — typically fast since initialisation is close to the optimum.
Implementation Pipeline Complete robust F estimation: (1) detect and match features (SIFT/ORB), (2) normalise correspondences (Hartley), (3) RANSAC with 8-point algorithm and Sampson inlier test, (4) refine on inliers with LM. This pipeline is implemented in OpenCV's findFundamentalMat() function with the FM_RANSAC flag.
