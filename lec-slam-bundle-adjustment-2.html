Bundle Adjustment II

Oct 2025 MSc Mobile Robotics, UniBonn

This lecture continues Bundle Adjustment with the practical aspects that make it work in real systems: gauge freedom, robust loss functions, incremental vs global strategies, and the Ceres Solver library that powers state-of-the-art SfM systems including COLMAP.

1. Gauge Freedom

The 3D reconstruction is defined only up to a 7-DOF similarity transform (3 translation, 3 rotation, 1 scale) — the "gauge." Any rigid-body transformation + rescaling of the entire scene plus cameras leaves all reprojection errors unchanged. This means the JᵀJ information matrix is rank-deficient (has a 7-dimensional null space), and the normal equations cannot be solved directly.

Three strategies to fix the gauge:

  • Fix the first camera: set P₁ = K[I|0], removing 6 DOF. Fix scale by setting ||t₂|| = 1.
  • Fix two cameras completely: removes 12 DOF (overconstrained by 5, but removes gauge).
  • Gauge penalty: add soft constraints penalising deviation from a reference pose — useful in incremental BA where fixing early cameras completely might degrade accuracy.

2. Robust Loss Functions

Standard squared reprojection error is very sensitive to outliers (wrong feature matches). A single bad match can corrupt many camera estimates. Replace the squared loss with a robust function ρ(r²) that down-weights large residuals:

  • Huber loss: ρ(r) = r² if |r| ≤ δ, else 2δ|r| − δ². Quadratic for small residuals, linear for large ones. Outliers contribute linearly rather than quadratically.
  • Cauchy loss: ρ(r) = log(1 + r²/σ²). Provides stronger down-weighting of large residuals — effectively ignores points with reprojection error much larger than σ.

These are implemented as Iteratively Reweighted Least Squares (IRLS): at each LM step, compute per-residual weights w(rᵢ) = ρ'(rᵢ²) / rᵢ and solve the weighted normal equations. Ceres Solver implements this transparently with its LossFunction interface.

3. Incremental vs Global Bundle Adjustment

Incremental BA (used in COLMAP) adds cameras one or a few at a time. After each addition, run local BA on a sliding window of recent cameras and their visible points. Periodically run global BA over the full reconstruction. This is efficient and robust — errors are caught early before they propagate. The key challenge is avoiding drift over long sequences.

Global BA optimises all cameras and points simultaneously. Yields the most accurate result but costs O(N³) in the number of cameras. Only practical for moderate-size reconstructions (hundreds of cameras) or with efficient iterative linear solvers.

4. Ceres Solver

Ceres (Agarwal et al., Google) is the industrial-standard library for BA. Key features:

  • Automatic differentiation: Ceres computes Jacobians automatically using dual numbers, eliminating hand-derivation of analytic Jacobians (though user-provided Jacobians are supported and faster).
  • Schur complement backend: the DENSE_SCHUR and SPARSE_SCHUR linear solvers implement the Schur complement trick automatically.
  • Iterative solvers: Conjugate Gradient (CG) with Jacobi or cluster Jacobi preconditioners scale to millions of points.
  • Robust kernels: built-in Huber, Cauchy, Arctan, and custom loss functions.

5. COLMAP

COLMAP (Schönberger & Frahm, 2016) is the state-of-the-art open-source SfM pipeline. It uses: SIFT features + GPU descriptor extraction, exhaustive/vocabulary-tree matching, geometric verification via RANSAC, incremental reconstruction, and Ceres-based BA. On large photo collections it produces dense reconstructions via MVS (Multi-View Stereo) as a second stage. It is the standard baseline comparison for any new SfM or visual SLAM method.

COLMAP on the Lab Machines COLMAP is installed on the lab machines at UniBonn. To reconstruct a scene from a folder of images: colmap automatic_reconstructor --workspace_path /path/to/ws --image_path /path/to/imgs. The sparse model can be visualised in the COLMAP GUI or exported for further processing.
