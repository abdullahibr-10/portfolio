Features — Descriptors

Nov 2025 MSc Mobile Robotics, UniBonn

A descriptor is a compact, distinctive representation of the local image patch around a keypoint. Good descriptors are invariant to the nuisances that change appearance across views (illumination, rotation, scale) while remaining discriminative enough to distinguish different scene points.

1. The Descriptor Goal

Given a keypoint at location (x, y) with scale σ and orientation θ, extract a canonical patch (normalised for scale and rotation), and encode its appearance as a fixed-length vector. Two descriptors from the same scene point should be close in descriptor space; descriptors from different points should be far apart. The challenge: images of the same scene point can differ significantly due to lighting, viewpoint angle, and partial occlusion.

2. SIFT Descriptor

The SIFT descriptor (Lowe, 2004) is the gold standard for accuracy:

  1. Extract a 16×16 pixel patch around the keypoint, scaled to the characteristic scale σ* and rotated to the dominant orientation.
  2. Divide into a 4×4 grid of cells (each 4×4 pixels).
  3. In each cell, compute an 8-bin histogram of gradient orientations, weighted by gradient magnitude and a Gaussian centered on the keypoint.
  4. Concatenate the 4×4×8 = 128 values into a vector.
  5. L2-normalise. Clamp values above 0.2 (reduces sensitivity to non-linear illumination changes). Renormalise.

Result: a 128-dimensional float vector. Matching with Euclidean distance in 128D gives extremely reliable correspondences at the cost of storage and matching speed.

3. Dominant Orientation for Rotation Invariance

Before computing the SIFT descriptor, find the dominant gradient orientation in the keypoint neighbourhood. Compute a 36-bin histogram of gradient orientations (weighted by magnitude and a Gaussian) in a window of radius 3σ* around the keypoint. The peak of this histogram is the dominant orientation. The patch is then rotated by this angle before description, making the descriptor rotation-invariant.

4. BRIEF Descriptor

BRIEF (Binary Robust Independent Elementary Features) trades accuracy for extreme compactness and speed. Pre-define 256 pairs of pixel locations (p_k, q_k) within the patch (sampled from a Gaussian distribution). The descriptor is a 256-bit binary string where bit k = 1 if I(p_k) > I(q_k). Matching is done with Hamming distance (XOR + popcount), which is a single CPU instruction on modern hardware. BRIEF alone is not rotation-invariant.

5. ORB — Oriented FAST + Rotated BRIEF

ORB (Rublee et al., 2011) is the standard descriptor for real-time SLAM:

  • Detection: FAST keypoints with Harris scoring to select the best N.
  • Orientation: intensity centroid method — the vector from the patch centroid to its intensity-weighted centroid defines the orientation.
  • Description: BRIEF applied to the rotation-corrected patch, using a pre-learned set of point pairs that are both discriminative and uncorrelated.

ORB is 2 orders of magnitude faster than SIFT with only modest accuracy loss. It powers ORB-SLAM2/3, the most widely used real-time visual SLAM system.

6. Descriptor Matching and Lowe's Ratio Test

Brute-force matching compares each descriptor in image 1 against all descriptors in image 2 (O(N²) comparisons). For SIFT, use FLANN (approximate nearest neighbours). For binary descriptors (ORB/BRIEF), use linear scan with XOR+popcount — even a database of 10k binary descriptors can be searched in microseconds.

Lowe's ratio test filters ambiguous matches: compute the two nearest neighbours d₁ ≤ d₂. Accept the match only if d₁/d₂ < 0.8. If the best match is only marginally better than the second-best, the keypoint is in an ambiguous region (repeated texture, homogeneous area) and the match is rejected. This test alone eliminates the majority of false matches.

Float vs Binary Descriptors Float descriptors (SIFT, 128×4 = 512 bytes per keypoint) provide higher accuracy for wide-baseline matching and complex lighting changes. Binary descriptors (ORB, 256 bits = 32 bytes) are 16× more compact and match orders of magnitude faster — ideal for real-time SLAM but may struggle with large viewpoint changes (>60°).
