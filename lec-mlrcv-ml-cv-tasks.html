ML for Computer Vision Tasks

Oct 2025 MSc Mobile Robotics, UniBonn

Before deep learning dominated computer vision, classical pipelines combined hand-crafted image features with the ML classifiers covered in previous lectures. Understanding these methods is important both historically and practically — they remain competitive for small datasets, low-power embedded systems, and interpretable applications.

1. The Classical CV + ML Pipeline

The standard recipe: Image → Feature Extraction → Feature Encoding → Classifier → Prediction. Feature extraction transforms raw pixels into a compact, semantically meaningful representation. The classifier (SVM, Random Forest, logistic regression) then operates in this feature space, not the raw pixel space. The quality of the features determines the ceiling on accuracy.

2. HOG — Histogram of Oriented Gradients

HOG (Dalal & Triggs, 2005) was the state-of-the-art pedestrian detector before CNNs. The algorithm:

  1. Gradient computation: convolve image with [-1, 0, 1] and [-1, 0, 1]ᵀ kernels to get Gx and Gy. Compute magnitude |G| = √(Gx² + Gy²) and orientation θ = arctan(Gy/Gx).
  2. Cell histograms: divide image into 8×8 pixel cells. In each cell, accumulate a 9-bin histogram of gradient orientations (0°–180° unsigned), weighted by gradient magnitude.
  3. Block normalisation: group 2×2 cells into overlapping blocks. Concatenate the four 9-bin histograms (36-dim vector) and L2-normalise. Normalisation gives contrast invariance.
  4. Descriptor: concatenate all block vectors across the detection window. For a 64×128 pixel window: (7×15) blocks × 36 = 3,780-dimensional vector.

3. Sliding Window Detection

To detect objects at unknown locations and scales, we use sliding window search: evaluate a classifier at every possible position and scale in the image. Typically an image pyramid is built by resizing the image by a factor (e.g., 0.85) at each level, and the detection window slides across each pyramid level with a step size (e.g., 8 pixels). This yields thousands of candidate windows, most of which contain background.

Non-Maximum Suppression (NMS) removes duplicate detections: among overlapping boxes with high scores, retain only the highest-scoring box. Boxes with IoU > threshold (e.g., 0.5) with a retained box are suppressed.

4. Bag of Visual Words

BoVW generalises the text retrieval concept to images. Steps:

  1. Descriptor extraction: extract local descriptors (SIFT) at keypoints or on a dense grid across all training images.
  2. Codebook construction: cluster all descriptors with k-Means into k visual words (the codebook).
  3. Encoding: for a new image, assign each descriptor to its nearest visual word. Represent the image as a histogram of visual word frequencies (k-dimensional vector).
  4. Classification: train an SVM (with χ² or histogram intersection kernel) on these k-dim histograms.

BoVW loses all spatial information. Spatial Pyramid Matching (Lazebnik et al., 2006) recovers some by concatenating histograms computed at multiple spatial grid resolutions.

5. Evaluation for Detection

Intersection over Union (IoU): measures overlap between predicted box B_p and ground-truth box B_gt:

IoU = Area(B_p ∩ B_gt) / Area(B_p ∪ B_gt)

A detection is a True Positive if IoU > threshold (commonly 0.5 for PASCAL VOC). By varying the classifier threshold we get a precision-recall curve. The Average Precision (AP) is the area under this curve. Mean AP (mAP) averages AP across all object categories.

6. Limitations and the Motivation for CNNs

Hand-crafted features have a fundamental limitation: the designer must anticipate which image properties are discriminative. HOG works for rigid upright objects but fails for deformable objects, unusual viewpoints, or images with strong illumination variation. The codebook in BoVW must be rebuilt for each new domain. There is no end-to-end optimisation — the features and classifier are trained separately, so errors in feature design cannot be recovered downstream.

Transition to Deep Learning The 2012 AlexNet result (top-5 error 15.3% vs. 26.2% for the best classical method on ImageNet) demonstrated that features learned directly from data via convolutional networks could dramatically outperform hand-crafted features.
