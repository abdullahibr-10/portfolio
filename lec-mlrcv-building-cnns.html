Building CNNs

June 2025 MSc Mobile Robotics, UniBonn
What this lecture is about You already know what a CNN is and how to train one. This lecture asks the more practical question: given a fixed parameter budget, how do you spend it well? It walks through the history of ImageNet competition winners (2012–2022) to extract the design principles that matter, then covers how to make networks efficient (MobileNet, EfficientNet), and finally how to prevent them from overfitting (regularisation + data augmentation).

1 · Architecture Evolution: ImageNet 2012–2022

The ImageNet Large Scale Visual Recognition Challenge (ILSVRC) drove a decade of innovation. Every major idea from this period is still found in modern robotics perception stacks. Below is the canonical progression — know every row in the exam table.

1.1 AlexNet (2012) — The starter pistol

AlexNet proved that deep CNNs trained on GPUs could crush hand-crafted features. Architecture: 5 conv layers + 3 fully-connected layers = 8 learnable layers, 60 M parameters (most of them in the FC layers). Input is 227×227×3 with an aggressive 11×11, stride 4 first convolution.

AlexNet's enabling tricks:
  • ReLU — used after every layer; trains 6× faster than tanh at that scale
  • Two-GPU split — fit the model across two GTX 580 s (3 GB each)
  • Dropout — applied in the FC layers with \(p = 0.5\) to combat the 60 M parameter overfitting risk
  • Local Response Normalisation — lateral inhibition (largely replaced by Batch Norm today)
  • Ensemble of 5 models → 16.4 % top-5 error

1.2 ZFNet (2013) — Refinement by visualisation

Zeiler & Fergus used deconvolutions to visualise what each filter was learning. They discovered that AlexNet's first conv layer learned mixed, aliased features due to the large 11×11 stride-4 kernel. Fixing it to 7×7 stride 4 and tuning channel counts gave 11.7 % — the same depth (8 layers) but better tuned. ZFNet matters more as a methodology (see what your network is doing) than as a new architecture.

1.3 VGGNet (2014) — The power of depth & simplicity

Karen Simonyan and Andrew Zisserman's insight: instead of large kernels, stack many small ones. A stack of two 3×3 convolutions has the same receptive field as one 5×5 but uses \(2 \times 3^2 = 18\) vs \(5^2 = 25\) weights per channel — and has an extra non-linearity.

VGG design rules — memorise these:
  1. Only use 3×3 convolutions (stride 1, padding 1 — spatial size unchanged)
  2. 2×2 max pooling (stride 2) to halve spatial dimensions
  3. When spatial size is halved, double the number of filters (64→128→256→512…)
  4. End with three FC layers; the first two use dropout

VGG-16 has 138 M parameters, VGG-19 has 144 M — almost all in the FC layers. Despite this, VGG's representation was so good that VGG features were the go-to transfer learning backbone until ResNet arrived. VGG required pre-training on a smaller dataset; training from scratch from random initialisation often failed at 16–19 layers.

1.4 GoogLeNet / Inception (2014) — Efficiency through branching

GoogLeNet achieved a better error rate (6.7 %) than VGG with 20× fewer parameters (~7 M). The key ideas:

1. The Inception layer. Rather than choosing which kernel size to use, use all of them in parallel and let the network learn which matters:

Inception layer = concat(1×1 conv, 3×3 conv, 5×5 conv, 3×3 max pool)

2. 1×1 bottleneck convolutions. Before each expensive 3×3 or 5×5 conv, apply a 1×1 conv to reduce channel depth. This dramatically cuts FLOPs. A 3×3 conv on 256 channels costs 256×3×3×256 weights; a 1×1 to 64 then 3×3 then 1×1 back to 256 costs only a fraction of that.

3. Global Average Pooling (GAP). Instead of flattening to FC layers, average each feature map spatially to a single scalar. If the last feature volume is \(H \times W \times C\), GAP gives a \(C\)-dimensional vector directly. This cuts millions of FC parameters and acts as a built-in regulariser.

4. Auxiliary classifiers. Two intermediate softmax heads are attached mid-network during training to push gradient deeper (pre-BatchNorm). They are removed at inference. With Batch Normalisation these are unnecessary.

1.5 ResNet (2015) — The single most important architecture idea Exam Focus

He et al. observed something counter-intuitive: when training a 56-layer plain network versus an 18-layer one, the deeper network had higher training error — not just test error. This is not overfitting; it is an optimisation problem.

The identity layer argument: Consider a shallow network that solves a task well. A deeper network should be at least as good — in the worst case, the extra layers could just learn the identity mapping and ignore the input. But in practice, SGD cannot find this solution through random initialisation in deep networks. ResNet makes it easy.

Skip connections (shortcut connections). Instead of learning the full mapping \(H(\mathbf{x})\), a residual block learns:

\[\mathbf{y} = F(\mathbf{x}, \{W_i\}) + \mathbf{x}\]

where \(F(\mathbf{x})\) is the residual: what the block needs to add on top of the identity. If the identity is the optimal mapping, the network simply learns \(F \to 0\). This is much easier to optimise than learning the identity through a stack of weight layers from scratch.

Basic block (ResNet-18, ResNet-34):

Input → Conv 3×3 → BN → ReLU → Conv 3×3 → BN → (+) → ReLU → Output
                                                                   ↑
                                                              skip: x (or 1×1 projection)

Bottleneck block (ResNet-50, -101, -152):

Input (256 ch) → 1×1 Conv (64 ch) → BN → ReLU
                           → 3×3 Conv (64 ch) → BN → ReLU
                           → 1×1 Conv (256 ch) → BN → (+) → ReLU → Output
                                                           ↑
                                                       skip (1×1 projection if dims differ)

The bottleneck reduces the expensive 3×3 computation by first projecting down to 64 channels, doing the spatial mixing, and projecting back up. This is what makes 50+ layer networks feasible.

ResNet details worth knowing:
  • Batch Norm after every Conv, before ReLU
  • Downsampling via strided conv (stride 2), not pooling
  • When the skip connection crosses a resolution/channel boundary, use a 1×1 conv projection on the skip path
  • ResNet-152 achieves 3.6 % top-5 error — a 4× improvement over AlexNet in 3 years
  • ResNet-50 is the de-facto baseline in robotics vision today

2 · Efficient Architectures: Robots Have Power Budgets

The ImageNet winners optimised for accuracy on a GPU cluster. Robotics requires models that run on embedded hardware (Jetson, mobile CPU, microcontroller). The following architectures are designed to be fast and small with minimal accuracy trade-off.

2.1 MobileNet (2017) — Depthwise Separable Convolutions Exam Focus

A standard 3×3 conv layer with \(C_{in}\) input channels and \(C_{out}\) output channels applies \(C_{out}\) filters of shape \(3 \times 3 \times C_{in}\) to every spatial location. The total multiplication cost per spatial location is:

\[\text{Standard conv: } K^2 \cdot C_{in} \cdot C_{out} \cdot H \cdot W\]

MobileNet splits this into two steps:

  1. Depthwise conv — apply one \(K \times K\) filter per input channel independently. This does spatial mixing but no cross-channel mixing. Cost: \(K^2 \cdot C_{in} \cdot H \cdot W\).
  2. Pointwise conv — apply a \(1 \times 1 \times C_{in}\) conv to mix channels across the \(C_{out}\) output channels. Cost: \(C_{in} \cdot C_{out} \cdot H \cdot W\).

The total cost and the ratio to standard conv:

\[\text{Depthwise sep.: } (K^2 + C_{out}) \cdot C_{in} \cdot H \cdot W\] \[\text{Ratio} = \frac{K^2 + C_{out}}{K^2 \cdot C_{out}} = \frac{1}{C_{out}} + \frac{1}{K^2} \approx \frac{1}{K^2}\]

For \(K=3\): ratio \(\approx 1/9\). 9× fewer multiplications for a small accuracy drop. In practice MobileNet achieves 70.6 % top-1 accuracy with 4.2 M parameters versus VGG's 71.5 % with 138 M parameters — roughly the same accuracy at 30× the parameter efficiency.

2.2 MobileNetV2 (2018) — Inverted Bottleneck (MBConv)

MobileNetV2 introduces the inverted bottleneck block, which goes wide-narrow-wide in a way that is the opposite of the ResNet bottleneck:

ResNet BottleneckMobileNetV2 MBConv
Patternwide → narrow → widenarrow → wide → narrow
Step 11×1 compress (÷4)1×1 expand (×6, expansion factor \(t\))
Step 23×3 conv (full depth)3×3 depthwise conv
Step 31×1 expand (×4)1×1 project back (no ReLU!)
Skip?Yes (projection if needed)Yes (only if stride=1 and same dims)

The ReLU is omitted after the final projection step because ReLU on narrow (bottleneck-output) channels destroys information. The wide depthwise layer has ReLU6 (clipped at 6) which is numerically stable for fixed-point inference. MobileNetV2 has 20 learnable layers; all depth-wise separable.

3 · Neural Architecture Search (NAS)

Instead of designing architectures by hand, NAS automates the search. Three components:

Search Space

Defines the possible architectures: which operations (conv, pooling, skip), how many blocks, how to connect them. Heavily determines the quality of the result.

Search Strategy

Reinforcement learning (reward = validation accuracy) or evolutionary / genetic algorithms. An RL controller samples candidate architectures and trains a child network to evaluate them.

Performance Estimation

Training every candidate to convergence is too expensive. Alternatives: short training runs + extrapolation, weight sharing (DARTS — one supernetwork), or zero-shot proxies.

Outcome

NAS often finds non-intuitive architectures (e.g., asymmetric filters, unusual skip patterns). In practice it is usually run with a hardware latency constraint as a secondary objective.

4 · EfficientNet (2019) — Compound Scaling Exam Focus

There are three ways to scale a CNN up from a baseline:

  • Width scaling — more channels per layer (more features, up to a point)
  • Depth scaling — more layers (larger receptive field, but vanishing gradients)
  • Resolution scaling — larger input images (more spatial detail)

Each individual dimension exhibits diminishing returns. Tan & Le (Google) showed that the three dimensions are complementary and should be scaled together. Given a resource budget \(\phi\), they scale with fixed ratios \(\alpha, \beta, \gamma\) derived by a small grid search:

\[\text{depth:} \; d = \alpha^\phi, \quad \text{width:} \; w = \beta^\phi, \quad \text{resolution:} \; r = \gamma^\phi\] \[\text{subject to: } \alpha \cdot \beta^2 \cdot \gamma^2 \approx 2 \quad (\text{FLOPs roughly double per } \phi)\]

EfficientNet-B0 (the baseline) was found by NAS. EfficientNet B1–B7 are obtained by compund-scaling B0 with increasing \(\phi\).

ModelTop-1 Acc.Params
ResNet-5076.0 %26 M
EfficientNet-B077.1 %5.3 M
EfficientNet-B483.0 %19 M
EfficientNet-B784.4 %66 M

EfficientNet-B0 already beats ResNet-50 at 5× fewer parameters. The larger variants set new records across many transfer learning tasks.

5 · ConvNeXt (2022) — Modernising the CNN

After Vision Transformers (ViT) started outperforming CNNs, Liu et al. asked: "What if we systematically modernised a ResNet-50 with the design decisions from ViT's training recipe?" The result, ConvNeXt, is a pure CNN that matches or beats ViT on ImageNet with the same training setup. The key changes (each improving accuracy step by step):

ChangeFrom (ResNet)To (ConvNeXt)
Training recipe90 epoch, standard300 epoch, AdamW, modern augmentation
Macro designUniform stage ratioStage ratio 3:3:9:3 (Swin-inspired)
Stem7×7 stride-2 + max pool4×4 non-overlapping (patchify)
Bottleneck1×1 → 3×3 → 1×1 (standard)Inverted bottleneck (MobileNetV2-style)
Kernel size3×3 in bottleneck neck7×7 depthwise (large receptive field)
ActivationReLU (all layers)GELU (one per block)
NormalisationBatch NormLayer Norm
DownsamplingStrided conv in blockSeparate 2×2 stride-2 LN layer

The large depthwise kernel (7×7) captures long-range context within the CNN framework, closing the gap with the attention mechanism in ViT. ConvNeXt-T matches Swin-T on ImageNet and runs faster on standard hardware. It is increasingly used as a backbone in robotics perception.

6 · Overfitting and the Generalisation Gap Exam Focus

After choosing an architecture, the practical challenge is making the trained model generalise to unseen data. Two failure modes:

  • Overfitting: training loss decreases but validation loss increases. The model memorises training data. Fix: reduce model capacity, add regularisation.
  • Generalisation gap: both losses converge, but validation loss is consistently higher than training loss. The gap measures how much is lost at deployment. Fix: data augmentation, label smoothing.

In the large-scale regime (ImageNet, fine-tuning on millions of images) the generalisation gap is what matters most. The following techniques all attack it.

6.1 Regularisation techniques

Early Stopping

Monitor validation loss every epoch. Stop training when it has not improved for \(k\) epochs. The model at the best validation epoch is saved. Simple and universally applicable — always use it.

L2-Regularisation / Weight Decay Key Equation

Add a penalty to the loss that discourages large weights:

\[L(\theta) = \frac{1}{N}\sum_{i=1}^{N} \ell\!\left(y_i,\, f(\mathbf{x}_i;\theta)\right) + \lambda\|\theta\|^2\]

Taking the gradient and plugging into SGD:

\[\theta_{k+1} = \theta_k - \eta\,\frac{\partial L}{\partial \theta} - \lambda\,\theta_k\]

The \(-\lambda\theta_k\) term decays (shrinks) the weights each step — hence "weight decay". The hyperparameter \(\lambda\) (typically \(10^{-4}\) to \(5\times10^{-4}\)) controls the trade-off.

L2 vs weight decay with Adam: With SGD, L2 regularisation and weight decay are mathematically equivalent. With Adam they are not — because Adam adapts the learning rate per parameter, the L2 gradient gets scaled differently from the direct weight decay. Always use AdamW (Adam with decoupled weight decay) when combining Adam with regularisation. Apply weight decay only to weights of conv and FC layers — not to biases, BatchNorm scale parameters (\(\gamma\)), or BatchNorm shift parameters (\(\beta\)).

Dropout

During training, randomly zero each activation with probability \(p\) (typically \(p = 0.1\)–\(0.5\)). This prevents individual neurons from becoming co-adapted — the network cannot rely on any single feature. At test time, multiply all activations by \((1-p)\) to maintain the expected output scale (or equivalently, scale during training — "inverted dropout").

In CNNs, dropout is usually applied only after the FC layers (or at the global pooling output), not inside conv stacks — conv layers are already regularised by weight sharing and BN.

Stochastic Depth

Huang et al. (2016) proposed randomly dropping entire ResNet blocks during training — replace the block output with just the skip connection \(\mathbf{y} = \mathbf{x}\). The drop probability increases linearly with depth: shallow blocks are always kept (they learn basic features); deeper blocks are more frequently dropped (they learn finer details that can be varied). A typical schedule: layer \(l\) of \(L\) total has survival probability \(1 - \frac{l}{L}(1-p_L)\) where \(p_L = 0.5\).

This also dramatically reduces training time (fewer blocks computed each forward pass) and is used in ConvNeXt, EfficientNet, and Vision Transformers.

Label Smoothing

Standard cross-entropy trains towards a one-hot target: 100 % confidence in one class. This makes the model overconfident and its predictions poorly calibrated. Label smoothing mixes the one-hot target with the uniform distribution:

\[\tilde{y}_k = (1 - \varepsilon)\,y_k + \frac{\varepsilon}{K}\]

where \(K\) is the number of classes and \(\varepsilon\) is small (typically 0.1). The model is now penalised for being too certain. This improves calibration and generalisation, especially when combined with mixup.

7 · Data Augmentation Exam Focus

Data augmentation attacks the generalisation gap from the data side: instead of changing the model, create more diverse training examples by transforming existing ones. The label (class) is assumed to be invariant to the transformation.

7.1 Standard augmentations

Horizontal flip

Mirror the image left-right. Label-invariant for most classification tasks (a dog is still a dog). Never apply vertical flip without justification — an upside-down dog is unusual.

Random crop & resize

Crop a random subregion and resize back to the input resolution. Teaches the network that the object can appear anywhere and at different scales. Standard ImageNet training crops 224×224 from 256×256 images.

Colour distortion

Randomly perturb brightness, contrast, saturation, hue. Teaches invariance to lighting conditions — critical for outdoor robotics.

7.2 RandAugment Key Method

Cubuk et al. (2020) sampled \(N\) augmentation policies from a fixed set (including shear, rotate, posterize, solarize, equalize, etc.) and applied them sequentially with a single global magnitude \(M\). The search space is a simple 2D grid: \((N, M)\) — sweep on a small validation set. This beats the complex per-layer policy search of AutoAugment at a fraction of the search cost.

RandAugment at a glance: Pick \(N\) operations randomly, apply each with magnitude \(M\). Tune \(N \in \{1,2,3\}\) and \(M \in \{5,...,30\}\) by grid search on validation.

7.3 Mixup Key Method

Zhang et al. (2018) create synthetic training samples by linearly interpolating pairs of training examples and their labels:

\[\tilde{\mathbf{x}} = \lambda\,\mathbf{x}_i + (1-\lambda)\,\mathbf{x}_j\] \[\tilde{y} = \lambda\,y_i + (1-\lambda)\,y_j, \quad \lambda \sim \text{Beta}(\alpha, \alpha)\]

The resulting image looks like a ghost-superposition of two examples. The model is trained to produce the interpolated label distribution. Mixup enforces linear behaviour between training examples and strongly regularises over-confidence.

7.4 Cutout / Random Erasing

DeVries & Taylor (2017) randomly select a rectangular region of an image and fill it with the channel mean (making it invisible/uninformative). This acts like dropout applied at input level: the network must classify using partial information, learning more distributed representations. Random Erasing is a variant that fills with random pixel values instead of the mean.

7.5 CutMix Key Method

Yun et al. (2019) combine Cutout and Mixup: instead of filling the erased region with noise, paste a crop from another image. Mix the labels by area proportion:

\[\tilde{\mathbf{x}} = \mathbf{M} \odot \mathbf{x}_i + (1 - \mathbf{M}) \odot \mathbf{x}_j\] \[\tilde{y} = \lambda\,y_i + (1-\lambda)\,y_j, \quad \lambda = \frac{\text{area of crop}}{H \times W}\]

where \(\mathbf{M}\) is a binary mask. CutMix outperforms Cutout and Mixup individually, especially for object detection tasks (where localisation matters). All pixels are from real data, unlike Mixup's ghost images.

8 · Practical Advice

8.1 What do modern models actually use?

Training recipes for state-of-the-art models combine many of the above techniques:

ModelAugmentationRegularisation
EfficientNet AutoAugment Stochastic Depth + Dropout
ConvNeXt Mixup + CutMix + RandAugment + Random Erasing Stochastic Depth + Label Smoothing + Weight Decay

8.2 Starting point recommendation

  1. Baseline: ResNet-50 pre-trained on ImageNet. Reliable, well-understood, fast to fine-tune.
  2. If you need better accuracy: EfficientNet-B4 or ConvNeXt-T. Similar parameter count, noticeably better performance.
  3. If you need to run on embedded hardware: MobileNetV2 or MobileNetV3.
  4. For everything else: do not train from scratch. Fine-tune ImageNet pre-trained weights.

8.3 Use TIMM

The pytorch-image-models (timm) library has over 700 pre-trained models with a unified API. Never implement these architectures from scratch for production use.

pip install timm

import timm
# List available models
timm.list_models('efficientnet*')

# Load pretrained ResNet-50
model = timm.create_model('resnet50', pretrained=True)

# Fine-tune: replace classifier head for your task
model = timm.create_model('resnet50', pretrained=True, num_classes=10)

# Get data config (normalisation, input size) matching the pretrained weights
data_config = timm.data.resolve_model_data_config(model)
transforms = timm.data.create_transform(**data_config, is_training=True)
Exam Cheat Sheet
  • AlexNet: 8 layers, 60 M params, ReLU + dropout, 16.4 %
  • VGG rules: 3×3 only; halve spatial → double filters; 138 M params; 7.3 %
  • GoogLeNet: Inception (parallel branches), 1×1 bottleneck, Global Avg Pooling replaces FC; 6.7 %
  • ResNet skip: \(\mathbf{y} = F(\mathbf{x}) + \mathbf{x}\); basic block = two 3×3; bottleneck = 1×1→3×3→1×1; 3.6 %
  • Depthwise sep.: \(\approx 1/K^2\) of standard conv cost (8–9× for 3×3)
  • MBConv: narrow → expand (×t) → depthwise → project → narrow (no ReLU on project)
  • EfficientNet compound scaling: scale depth \(\times\) width \(\times\) resolution together; ratio \(\alpha\beta^2\gamma^2 \approx 2\)
  • ConvNeXt: 7×7 depthwise, GELU, LayerNorm, inverted bottleneck — pure CNN matching ViT
  • Weight decay: \(\theta_{k+1} = \theta_k - \eta\partial L/\partial\theta - \lambda\theta_k\); use AdamW (not Adam + L2)
  • Label smoothing: \(\tilde{y}_k = (1-\varepsilon)y_k + \varepsilon/K\)
  • Mixup: mix images AND labels linearly; \(\lambda \sim \text{Beta}(\alpha,\alpha)\)
  • CutMix: paste a crop; label proportion = crop area / total area; best for detection
  • Stochastic Depth: drop entire blocks (keep skip); drop probability increases with depth
