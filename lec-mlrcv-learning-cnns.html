Learning CNNs

Nov 2025 MSc Mobile Robotics, UniBonn

Designing an architecture is only half the work. This lecture covers how CNNs are trained: backpropagation through convolutional layers, initialisation strategies, and regularisation and optimisation methods that make modern deep networks feasible to train.

1. Loss Functions

For multi-class classification with a softmax output layer, the standard loss is cross-entropy: L = −Σₖ yₖ log(ŷₖ) where y is the one-hot target and ŷ is the softmax output. This is equivalent to maximising the log-likelihood of the true class. For regression tasks (e.g., bounding box coordinates, pose estimation), Mean Squared Error L = (1/n) Σᵢ ||yᵢ − ŷᵢ||² is typical, though smooth L1 (Huber) loss is preferred when outliers are present.

2. Backpropagation Through Conv Layers

Backprop computes the gradient of the loss with respect to all parameters by the chain rule. For a conv layer with input X and kernel K producing output Y = X ★ K:

  • Gradient w.r.t. kernel K: ∂L/∂K = X ★ δ, where δ = ∂L/∂Y is the upstream gradient. This is a cross-correlation of the input with the upstream gradient — the same sliding operation as the forward pass.
  • Gradient w.r.t. input X (to pass to previous layer): ∂L/∂X = δ ★★ K, a full convolution (with padding) of the upstream gradient with the flipped kernel.

Modern frameworks (PyTorch, TensorFlow) implement these as highly optimised CUDA kernels. Understanding the derivation helps debug shape errors and design custom layers.

3. Weight Initialisation

Initialising all weights to zero causes all neurons to compute the same gradient — the symmetry problem. All neurons remain identical after any number of updates. Instead:

  • Xavier (Glorot) initialisation: W ~ Uniform(−√(6/(nᵢₙ+nₒᵤₜ)), √(6/(nᵢₙ+nₒᵤₜ))). Designed to keep variance of activations constant through layers with tanh/sigmoid. Standard deviation: σ = √(1/nᵢₙ).
  • He initialisation: W ~ N(0, √(2/nᵢₙ)). Accounts for ReLU zeroing half the inputs; recommended for ReLU networks. Without this, variance shrinks layer by layer, leading to vanishing gradients in deep nets.

4. Batch Normalisation

Batch Norm (Ioffe & Szegedy, 2015) normalises the pre-activation of each layer within a mini-batch of size m:

μ_B = (1/m) Σᵢ xᵢ, σ²_B = (1/m) Σᵢ (xᵢ − μ_B)²
x̂ᵢ = (xᵢ − μ_B) / √(σ²_B + ε), yᵢ = γ x̂ᵢ + β

Learnable parameters γ (scale) and β (shift) allow the network to recover any desired distribution if needed. Benefits: (1) reduces internal covariate shift — each layer sees more stable input distributions; (2) allows higher learning rates; (3) acts as a mild regulariser (adds noise through batch statistics); (4) reduces sensitivity to initialisation. At test time, μ_B and σ²_B are replaced with running averages collected during training.

5. Dropout

Dropout (Srivastava et al., 2014) randomly sets each neuron's activation to 0 with probability p during training (typically p = 0.5 for FC layers, p = 0.1–0.2 for conv layers). At test time, no dropout is applied but activations are multiplied by (1−p) to maintain the expected sum. Dropout can be interpreted as training an ensemble of 2ⁿ sub-networks (for n neurons) — at test time all models are averaged in a single forward pass. It is most effective in FC layers; BatchNorm has largely replaced it in conv layers.

6. Optimisers

Beyond vanilla SGD (θ ← θ − α∇L), several improved optimisers are standard:

  • SGD + Momentum: accumulate a velocity vector v ← β·v − α∇L, then θ ← θ + v. Momentum (β ≈ 0.9) accelerates through consistent gradient directions and dampens oscillations.
  • RMSProp: maintain a running average of squared gradients E[g²]ₜ = β·E[g²]_{t-1} + (1−β)·g²ₜ. Update: θ ← θ − α·gₜ / √(E[g²]ₜ + ε). Adapts learning rate per parameter.
  • Adam: combines momentum and per-parameter adaptive rates. Maintains first moment mₜ = β₁mₜ₋₁ + (1−β₁)gₜ and second moment vₜ = β₂vₜ₋₁ + (1−β₂)gₜ². Bias-corrected: m̂ₜ = mₜ/(1−β₁ᵗ), v̂ₜ = vₜ/(1−β₂ᵗ). Update: θ ← θ − α·m̂ₜ/(√v̂ₜ + ε). Default hyperparameters β₁=0.9, β₂=0.999, ε=10⁻⁸, α=0.001 work well across many tasks.
Practical Note Adam converges quickly but sometimes generalises slightly worse than SGD with momentum + cosine annealing. A common strategy: use Adam for fast initial convergence, then switch to SGD + momentum for the final training phase to squeeze out extra accuracy.
