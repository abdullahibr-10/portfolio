Learning CNNs

June 2026 MSc Mobile Robotics, UniBonn

This lecture establishes the mathematical foundations and engineering mechanics of training Convolutional Neural Networks (CNNs) for visual perception in robotics. By the end of these notes, you will completely understand how parameter learning is framed as continuous empirical loss minimization, how gradient flow is orchestrated across modular compute graphs via backpropagation, the mechanical trade-offs of modern stochastic optimizers, and the crucial stabilization roles played by weight initialization and layer normalization.

1 · Loss Functions & Empirical Objective Minimization

To teach a neural network architecture to perceive its environment, we must define a quantifiable scalar metric that measures the divergence between our model's predicted hypothesis and the actual ground truth targets. Formally, for a training dataset containing \( N \) independent samples, the global empirical objective function \( L(\theta) \) parameterized by the network weights \( \theta \) is formulated as:

\[ L(\theta) = \frac{1}{N} \sum_{i=1}^{N} l(y_i, f(x_i; \theta)) \]

Here, \( l(y_i, f(x_i; \theta)) \in \mathbb{R} \) represents the individual sample loss evaluating the exact scalar penalty between the target label \( y_i \) and the network prediction \( f(x_i; \theta) \). Depending on whether our robotic perception task dictates continuous coordinate regression or discrete semantic classification, we rely on two primary loss formulations:

L2 Loss (Mean Squared Error)

For regression tasks—such as predicting continuous robot end-effector velocities or visual bounding box coordinate offsets—we rely on the L2 loss. The individual sample penalty is defined as:

\[ l(y_i, f(x_i)) = (y_i - f(x_i; \theta))^2 \]

Intuitively, the quadratic penalty forces predicted continuous values \( f(x) \) to converge tightly toward target values \( y \). From a strict probabilistic standpoint, minimizing the L2 loss is mathematically equivalent to maximizing the negative log-likelihood of the observed data assuming a normally distributed zero-mean Gaussian error. In PyTorch, this is instantiated via torch.nn.MSELoss. When evaluating a mini-batch of size \( N \), the unreduced batch loss vector is represented as \( l(x, y) = L = \{l_1, \dots, l_N\} \), where each element is \( l_n = (x_n - y_n)^2 \).

Cross-Entropy Loss

For multi-class semantic classification (e.g., categorizing an image as an automobile, truck, or dog), we evaluate categorical probability distributions. Given raw unnormalized network output scores (logits) \( f_1(x), \dots, f_C(x) \) spanning \( C \) distinct classes, we map them into valid probability space using a Softmax activation:

\[ P(y=j|x) = \text{softmax}_j(f_1(x), \dots, f_C(x)) = \frac{\exp(f_j(x))}{\sum_{k} \exp(f_k(x))} \]

Evaluating the negative log-likelihood (NLL) for the correct target class index \( j \) yields our standard classification loss:

\[ l(j, f(x)) = -\log \frac{\exp(f_j(x))}{\sum_{k} \exp(f_k(x))} = -f_j(x) + \log\left(\sum_{k} \exp(f_k(x))\right) \]

To understand why this negative log-likelihood represents "cross-entropy", recall information theory. The cross-entropy \( H(P, Q) \) between a true target distribution \( P \) and an estimated network distribution \( Q \) is defined as:

\[ H(P, Q) = -\sum_{k} P(x_k) \log Q(x_k) \]

In standard classification, our target label \( j \) is represented as a deterministic one-hot distribution vector \( P(x) = (0, \dots, 0, 1, 0, \dots, 0) \) where only the \( j \)-th entry equals 1. Substituting this one-hot \( P(x) \) and our Softmax hypothesis \( Q(x) \) into the definition collapses the summation entirely to \( -1 \log Q(x_j) \), perfectly matching our negative log-likelihood derivation. Note that \( P(x) \) can also accommodate soft target distributions, such as those utilized in label smoothing.

Exam Pitfall · PyTorch Cross-Entropy Implementation: In PyTorch, the criterion torch.nn.CrossEntropyLoss combines LogSoftmax and NLLLoss into one single highly optimized numerical class. Consequently, you must feed it raw, unnormalized logits rather than post-softmax probabilities. It expects target class indices strictly as integers within the range \( [0, C-1] \). Furthermore, it accepts an optional 1D weight tensor to rebalance gradient penalties across highly unbalanced training sets.

2 · Backpropagation & Compute Graphs

Once our scalar objective \( L(\theta) \) is established, gradient-based optimization dictates that we compute its partial derivatives to determine the precise direction in which to adjust our parameter weights to minimize loss. Because deep neural networks are deeply nested compositions of non-linear functions—e.g., \( L(\theta_1, \theta_2, \theta_3) = f_3(f_2(f_1(x; \theta_1); \theta_2); \theta_3) \)—we compute these derivatives systematically using the multivariate chain rule.

\[ \frac{\partial L}{\partial \theta_3} = \frac{\partial L}{\partial f_3} \frac{\partial f_3}{\partial \theta_3}, \quad \frac{\partial L}{\partial \theta_2} = \frac{\partial L}{\partial f_3} \left[ \frac{\partial f_3}{\partial f_2} \frac{\partial f_2}{\partial \theta_2} \right], \quad \frac{\partial L}{\partial \theta_1} = \frac{\partial L}{\partial f_3} \frac{\partial f_3}{\partial f_2} \left[ \frac{\partial f_2}{\partial f_1} \frac{\partial f_1}{\partial \theta_1} \right] \]

Compute Graphs & Local Connectivity

To resolve the complex order of computation, we compile operational layers into a directed graph. In a compute graph, nodes encode distinct mathematical operations, while directed edges determine information flow and operational dependencies. Active training proceeds across two sequential phases:

  • Forward Pass: We traverse the directed graph from inputs to outputs, evaluating and caching all intermediate operational activations.
  • Backward Pass: We traverse backward from the loss, passing gradient "messages" along the edges via the chain rule to reuse calculated derivatives.

The fundamental elegance of backpropagation lies in local connectivity. An individual compute node does not need to understand the global network architecture; computing its derivatives involves only local information. For an arbitrary operation \( z = f(x, y) \), the local backward pass is governed by a universal operational product:

\[ \text{downstream gradient} = \text{upstream gradient} \times \text{local gradient} \]

Mathematically, if the upstream gradient arriving at the node's output is \( \frac{\partial L}{\partial z} \), the node simply multiplies it by its own analytic local partial derivatives \( \frac{\partial z}{\partial x} \) and \( \frac{\partial z}{\partial y} \) to calculate downstream gradients \( \frac{\partial L}{\partial x} \) and \( \frac{\partial L}{\partial y} \) backward along its incident edges.

Vector Backpropagation & Jacobians

When transitioning from scalar nodes to multi-dimensional tensor transformations, our local derivatives expand into matrices. For a layer mapping an input vector \( x \in \mathbb{R}^N \) to an output vector \( f(x) \in \mathbb{R}^M \), the local derivative is the full Jacobian matrix \( J \in \mathbb{R}^{M \times N} \) containing all first-order partial derivatives:

\[ \frac{df}{dx} = \begin{pmatrix} \frac{\partial f_1}{\partial x_1} & \dots & \frac{\partial f_1}{\partial x_N} \\ \vdots & \ddots & \vdots \\ \frac{\partial f_M}{\partial x_1} & \dots & \frac{\partial f_M}{\partial x_N} \end{pmatrix} \]

If our node processes an input tensor \( x \in \mathbb{R}^{D_x} \) and weight parameter \( y \in \mathbb{R}^{D_y} \) to yield \( z \in \mathbb{R}^{D_z} \), receiving an upstream gradient vector \( \frac{\partial L}{\partial z} \in \mathbb{R}^{1 \times D_z} \), the downstream gradients are computed via matrix multiplication with the local Jacobians \( \frac{\partial z}{\partial x} \in \mathbb{R}^{D_z \times D_x} \) and \( \frac{\partial z}{\partial y} \in \mathbb{R}^{D_z \times D_y} \).

Mechanics of ReLU & Memory Overhead

Consider the element-wise Rectified Linear Unit activation \( f(x) = \max(0, x) \). Analyzing its downstream gradient vector shows that it acts as a dynamic conditional switch:

\[ \frac{\partial L}{\partial x_i} = \begin{cases} \frac{\partial L}{\partial y_i}, & \text{if } x_i > 0 \\ 0, & \text{otherwise} \end{cases} \]

Consequently, ReLU actively "blocks" partial derivatives from flowing backward through compute nodes where the forward activation was zero or negative. Notice that while the theoretical Jacobian of a fully connected layer or ReLU is a massive matrix, deep learning frameworks do not construct these matrices explicitly in memory; instead, they evaluate the vector-Jacobian products dynamically on the fly.

Hardware & Framework Insight: In PyTorch custom torch.autograd.Function classes, intermediate tensors needed to evaluate analytic local gradients are explicitly cached during the forward pass via ctx.save_for_backward(x, y) and retrieved during the backward pass via ctx.saved_tensors. Because the forward pass must cache these intermediate activations across the entire compute graph, training a deep network requires roughly an additional model size in active GPU memory compared to pure test-time inference. To prevent out-of-memory errors in massive architectures, we apply gradient checkpointing, discarding intermediate activations and storing only "heavy" forward passes, re-evaluating intermediate steps dynamically on the fly during the backward pass.

3 · Optimization Algorithms

With parameter gradients calculated, we must apply an iterative optimization method to step down the loss surface. Evaluating the exact analytic gradient \( \frac{dL}{d\theta} \) over all data (\( N \approx 10^6 \)) at every step is computationally intractable. Instead, Stochastic Gradient Descent (SGD) computes many small inaccurate updates using a small mini-batch of examples (e.g., \( N = 32, 64, 128 \)) randomly sampled from all training data.

\[ \theta_t = \theta_{t-1} - \eta \frac{dL}{d\theta} \]

Here, \( \eta \) represents the hyperparameter learning rate. Standard SGD exhibits slow convergence in anisotropic ravines—where the loss surface curves much more steeply in one dimension than another—resulting in oscillations across the valley and slow progress along the floor. Saddle points can stall learning entirely.

Momentum & Nesterov Acceleration

To dampen oscillations and build physical inertia across flat plateaus or saddle points, Polyak (1964) introduced Momentum. We maintain a velocity vector \( v_t \) that computes a moving average over past updates and the current gradient, dampened by a friction parameter \( \gamma \) (typically 0.9):

\[ v_t = \gamma v_{t-1} + \eta \frac{dL}{d\theta}, \quad \theta_t = \theta_{t-1} - v_t \]

Nesterov Accelerated Gradient (NAG, 1983) resolves overshooting by evaluating the gradient not at the current position \( \theta \), but at a projected "look-ahead position" \( \theta - \gamma v_{t-1} \). By accounting for future change ahead of time, NAG applies a braking correction before overshooting the minimum:

\[ v_t = \gamma v_{t-1} + \eta \frac{dL}{d(\theta - \gamma v_{t-1})}, \quad \theta_t = \theta_{t-1} - v_t \]

Adaptive Learning Rates: Adagrad & RMSProp

Standard SGD and Momentum apply the same global learning rate \( \eta \) to all parameters simultaneously. Duchi et al. (2011) proposed Adagrad, adjusting the learning rate depending on accumulated squared gradients \( G_t \):

\[ G_t = G_{t-1} + \left(\frac{dL}{d\theta}\right)^2, \quad \theta_t = \theta_{t-1} - \frac{\eta}{\sqrt{G_t + \epsilon}} \frac{dL}{d\theta} \]

Because Adagrad monotonically sums squared gradients, \( G_t \) only increases — leading to a vanishing learning rate that eventually stops learning. Tieleman & Hinton (2012) resolved this via RMSProp, replacing the monotonic summation with an exponentially decaying moving average (decay factor 0.9):

\[ G_t = 0.9\, G_{t-1} + 0.1 \left(\frac{dL}{d\theta}\right)^2, \quad \theta_t = \theta_{t-1} - \frac{\eta}{\sqrt{G_t + \epsilon}} \frac{dL}{d\theta} \]

Adam

Kingma & Ba (2015) formulated Adam (Adaptive Moment Estimation), combining momentum and parameter-wise adaptive learning rates. Adam maintains moving averages of past gradients (\( m_t \)) and past squared gradients (\( v_t \)):

\[ m_t = \beta_1 m_{t-1} + (1 - \beta_1) \frac{dL}{d\theta}, \quad v_t = \beta_2 v_{t-1} + (1 - \beta_2) \left(\frac{dL}{d\theta}\right)^2 \]

Because \( m_0 \) and \( v_0 \) are zero-initialized, their estimates are biased toward zero in early iterations. Adam introduces bias-correction terms before executing the parameter update:

\[ \hat{m}_t = \frac{m_t}{1 - \beta_1^t}, \quad \hat{v}_t = \frac{v_t}{1 - \beta_2^t}, \quad \theta_t = \theta_{t-1} - \frac{\eta}{\sqrt{\hat{v}_t} + \epsilon}\, \hat{m}_t \]

Standard defaults: \( \beta_1 = 0.9 \), \( \beta_2 = 0.999 \), \( \epsilon = 10^{-8} \).

The Practical Optimizer Recipe: Large-scale benchmarking (Schmidt et al., 2021) reveals that there is generally no clear winner across all tasks and datasets. The suggested recipe: run a set of optimizers with default settings and pick the best on validation. Adam converges quickly but sometimes generalises slightly worse than SGD + Momentum + cosine annealing for final fine-tuning.

4 · Initialization & Normalization

Even with sophisticated optimizers, training deep networks can suffer from vanishing or exploding gradients. As activations and gradients multiply across consecutive layers, signals can either grow exponentially to infinity or collapse entirely to zero. Two structural solutions: careful initialization and normalization of layer outputs.

Weight Variance & Xavier / He Initialization

Early architectures like AlexNet initialized weights from a naive Gaussian \( w \sim \mathcal{N}(0, 0.01) \). If the standard deviation is too small, activations collapse; if too large, they saturate.

Glorot & Bengio (2010) formulated Xavier Initialization to preserve unit variance across linear transformations. For a layer with \( D_{in} \) input units:

\[ w \sim \mathcal{N}\!\left(0,\; \frac{1}{\sqrt{D_{in}}}\right) \]

For convolutions, \( D_{in} = K \cdot K \cdot C_{in} \). Xavier works well for symmetric activations (tanh, sigmoid) but not for ReLU, because ReLU actively zeroes out half the domain — variance is halved at every successive layer.

He et al. (2015) introduced He Initialization, adding a \( \sqrt{2} \) gain factor to compensate for the halved ReLU variance:

\[ w \sim \mathcal{N}\!\left(0,\; \sqrt{2} \cdot \frac{1}{\sqrt{D_{in}}}\right) \]

This gain correction preserves active representation energy across arbitrarily deep ReLU networks.

Batch Normalization

Ioffe & Szegedy (2015) proposed Batch Normalization (BN) to counter internal covariate shift. For feature channel \( j \) over a mini-batch of \( N \) samples, BN computes channel-wise mean and variance:

\[ \mu_j = \frac{1}{N} \sum_{i=1}^{N} x_{i,j}, \quad \sigma_j^2 = \frac{1}{N} \sum_{i=1}^{N} (x_{i,j} - \mu_j)^2 \]

It then normalizes activations and applies learnable affine parameters \( \gamma \) (scale) and \( \beta \) (shift):

\[ \hat{x}_{i,j} = \frac{x_{i,j} - \mu_j}{\sqrt{\sigma_j^2 + \epsilon}}\,\gamma + \beta \]

The \( \epsilon \) prevents division by zero. Why learnable \( \gamma \) and \( \beta \)? If BN strictly forced zero mean and unit variance, it would destroy the network's expressive range. The learnable parameters allow the network to recover whatever distribution optimizes the downstream objective.

During training, BN uses live mini-batch statistics. At test time, it uses running averages accumulated during training (exponential momentum 0.1):

\[ \bar{\mu}_j \leftarrow 0.9\,\bar{\mu}_j + 0.1\,\mu_j, \quad \bar{\sigma}_j^2 \leftarrow 0.9\,\bar{\sigma}_j^2 + 0.1\,\sigma_j^2 \]

Architecturally, pre-activation (Conv → BN → ReLU) is most common, but post-activation (Conv → ReLU → BN) also works — empirically there is little difference.

Group Normalization

Batch Normalization's stability depends entirely on mini-batch size \( N \). High-resolution robotics tasks force hardware memory constraints to restrict \( N \) to very small values (2–8). With such small batches, BN statistics become noisy and validation error skyrockets.

Wu & He (2018) formulated Group Normalization (GN) to eliminate batch-dimension dependency. Instead of computing moments across the batch axis, GN divides feature channels into sub-groups and computes moments strictly across the spatial \( H, W \) dimensions within each group. GN statistics are independent of \( N \), so Group Norm maintains stable behaviour even when \( N = 2 \). Other variants: Layer Norm (normalise across \( C, H, W \)), Instance Norm (normalise across \( H, W \) per channel independently).

Exam Cheat Sheet · Learning CNNs
  • Loss: L2 penalizes squared errors for regression; Cross-Entropy = negative log-likelihood over Softmax for classification.
  • PyTorch CrossEntropyLoss: expects raw logits (fuses LogSoftmax + NLLLoss); targets are integer class indices in \([0, C-1]\).
  • Backprop: downstream gradient = upstream gradient × local gradient. ReLU blocks gradients wherever forward input ≤ 0.
  • Jacobians: multi-dimensional layers have Jacobian matrices; VJP evaluated on the fly, not materialized in memory.
  • Memory: caching activations for backward ≈ doubles GPU memory vs inference. Use gradient checkpointing for very deep nets.
  • Optimizers in order: SGD → Momentum (velocity, \(\gamma=0.9\)) → NAG (look-ahead gradient) → Adagrad (monotonic \(G_t\), stalls) → RMSProp (decaying \(G_t\)) → Adam (both + bias correction).
  • Adam defaults: \(\beta_1=0.9\), \(\beta_2=0.999\), \(\epsilon=10^{-8}\). Use AdamW (decoupled weight decay) when regularising.
  • Xavier init: \(\mathcal{N}(0, 1/\sqrt{D_{in}})\) — for linear / tanh layers.
  • He init: \(\mathcal{N}(0, \sqrt{2}/\sqrt{D_{in}})\) — for ReLU layers; compensates for zeroed half-domain.
  • Batch Norm: normalises per channel across batch \(N\); running averages used at test time; unstable at small \(N\).
  • Group Norm: normalises per channel-group across \(H,W\); batch-size independent; preferred for small-batch robotics tasks.
